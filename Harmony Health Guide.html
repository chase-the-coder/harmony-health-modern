Your editing guide.
Everything you need to update the site.

Edit any page on yourharmonyhealth.com by chatting with Claude in plain English. No coding knowledge required. When you're done, say "publish" and it goes live in about 90 seconds.

The most important habit

Confused by anything? Take a screenshot and give it to Claude.

If the screen looks different from this guide, a button isn't where you expect, an error pops up, or you just don't know what something means — don't guess. Claude can see images.

Press ⌘ + ⇧ + Ctrl + 4 to copy a screenshot to the clipboard. Paste it (⌘ + V) into the Claude chat, and ask "What am I looking at?" or "What should I click?"

Works every time. Use it whenever you're stuck.

The 3 steps, every time

  1. Double-click "Harmony Health editing" on your Desktop.

    VS Code opens to your project folder.

  2. Click the Claude icon in the left sidebar.

    It looks like a small chat bubble. The chat panel opens on the right.

  3. Tell Claude what you want, in plain English.

    Describe the change. Attach photos by dragging them in. When you're done, say "publish" or "ship it". Your changes go live in 90 seconds.

The master prompt

Paste this into Claude at the start of every new session. Press the copy button → click into the Claude chat panel → press ⌘ + V → press Return.

You are helping me edit the Harmony Health website. I am not a developer. Rules: 1. Ask me what I want to change before doing anything. 2. Make only the change I ask for. Don't "improve" other things. 3. If I describe something vague, ask me one clarifying question first. 4. If I attach a photo, ask where it should go before using it. 5. After the edit, explain what you changed in plain English (not code). 6. When I say "publish", "ship it", "deploy", or "go live", commit and push to main so Vercel deploys it. 7. Never invent medical statistics, patient counts, or clinical claims. If I ask you to add a stat, ask me for a source first. 8. If a build looks broken or risky, stop and ask me before pushing. 9. Keep responses short and friendly. No jargon. Now ask me: "What would you like to change today?"

Attaching a photo

Drag and drop straight into Claude.

If you want to replace a photo on the site, drag the image file from your Mac's Finder or Desktop directly into the Claude panel at the bottom of VS Code. Claude will tell you the file was attached, then ask where it should go.

Tell it plainly, like: "Put this on the About page as the main Megan photo" or "Replace the hero image on the Peptide Therapy page with this."

Publishing — how "ship it" works

When you say "publish", "ship it", "deploy", or "go live", Claude will:

  1. Save your changes to the project's history (a "commit")
  2. Upload the change to GitHub (a "push")
  3. Vercel automatically detects the push and rebuilds the site
  4. Your change is live at yourharmonyhealth.com in ~90 seconds

Open the site in a fresh browser window to see the update. If it hasn't changed yet, wait another minute and refresh.

If a change doesn't appear after 3-4 minutes — don't panic and don't tell Claude to "re-deploy" or "try again." Text Chase. He can check what happened in under a minute.

When something goes wrong

If Claude says something confusing, or you can't find what it's describing

Take a screenshot and show it to Claude.

  1. Press ⌘ + ⇧ + 4
  2. Drag a box around the thing you want to show
  3. The screenshot saves to your Desktop
  4. Drag it into the Claude panel, and say "I'm stuck on this — can you help?"

If a site is broken after publishing

Vercel keeps the last working version of your site live while it builds the new one. If the new build fails, the site doesn't break — it just keeps serving the last good version.

If the site does look wrong, don't have Claude keep editing to fix it. Text Chase — he can restore the previous version in under a minute.

If the desktop shortcut doesn't work

Make sure VS Code is in your Applications folder. Then try opening VS Code manually: ⌘ + Space, type "VS Code", press Return. From the File menu, choose Open Recent → harmony-health.

When to ask Chase instead of Claude

For these, text Chase first — Claude can handle them but the risk is higher if something goes wrong:

  • Adding an entirely new service page (not editing an existing one)
  • Changing the overall color or visual style of the site
  • Anything involving the domain (yourharmonyhealth.com) itself
  • Changing the way the booking link works
  • Removing whole sections
  • Any time Claude says "this might break something" or asks you to confirm twice
