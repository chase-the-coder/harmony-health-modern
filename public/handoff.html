For Chase — during the Zoom handoff call

Harmony Health handoff.
Everything you click, copy, and read.

Before the call, ZIP the project folder and send it to them. On the Zoom, work through the 8 parts below in order (~45 min). They interact entirely through the VS Code Claude extension's chat panel — Claude runs any setup commands for them, asks for approval before running each one, and explains what it's doing. The team member never types a shell command themselves. Every URL and prompt has a Copy button. Click once, paste on the call.

Quick alternative

One-command setup script

This script handles Parts 2, 3, and 6 automatically (VS Code, Claude extension, GitHub CLI, repo clone, npm install, Desktop shortcut). Have them open Terminal and paste:

curl -fsSL https://harmony-health-modern.vercel.app/setup.sh | bash

They still need to create accounts (Part 1) and connect Vercel + GoDaddy (Parts 4-5) manually.

2 Install VS Code + Claude extension (10 min)no terminal

2.1 VS Code

  1. Download:
    https://code.visualstudio.com/Download
  2. Click the big Mac Universal button
  3. Open the .zip — it extracts Visual Studio Code.app
  4. Drag Visual Studio Code.app into the Applications folder. Not optional. Desktop installs break settings persistence.
  5. Open VS Code (Spotlight → type "VS Code" → Enter)
  6. If macOS says "Are you sure you want to open it?" click Open

2.2 Install the Claude extension (all clicks, no typing)

  1. In VS Code, click the Extensions icon in the left sidebar (it looks like four little squares)
  2. In the search box at the top, type:
    Claude Code
  3. Find the one published by Anthropic (has a blue verified checkmark)
  4. Click Install
  5. When installed, you'll see a new Claude icon in the left sidebar (looks like a little chat bubble)

2.3 Sign in to Claude

  1. Click the new Claude icon in VS Code's left sidebar
  2. A panel opens on the right. Click Sign In.
  3. Browser opens to Claude's auth page. Log in with their Pro account.
  4. Browser says "You're logged in." The VS Code panel now shows a chat box.
You're done with setup typing From here on, everything happens in a chat window. Sign-in persists forever on this Mac — they'll never have to sign in again unless they get a new machine.
Tell them this now — it's the most important habit

Any time something looks confusing, take a screenshot and give it to Claude.

Claude can see images. If the UI looks different from this guide, a button isn't where you expect, an error pops up, or you just don't know what something means — don't guess and don't panic.

Press ⌘ + ⇧ + Ctrl + 4 to copy a screenshot to the clipboard, then paste it (⌘ + V) into the Claude chat, and ask "What am I looking at?" or "What should I click?"

Or press ⌘ + ⇧ + 4 to save to Desktop, then drag the image into the Claude chat. Either way works — Claude will read the screenshot and tell you what to do.

3 Unzip + let Claude set up GitHub (10 min)no commands from them

3.1 Unzip the folder Chase sent

  1. Chase sends harmony-health-modern.zip via Dropbox / WeTransfer / AirDrop
  2. They download to their Downloads folder
  3. Double-click the zip — creates a harmony-health-modern folder
  4. In Finder, drag that folder to their home folder and rename it to harmony-health

3.2 Open the folder in VS Code

  1. In VS Code: File → Open Folder
  2. Pick harmony-health in their home folder
  3. Click Yes, I trust the authors when prompted
  4. All the site files appear in the left sidebar

3.3 Open the Claude panel and paste the setup prompt

  1. Click the Claude icon in the left sidebar (chat bubble)
  2. In the chat box at the bottom, paste this prompt and press Return:
I just received this project from Chase. This is my very first time setting it up on my Mac. I'm not technical — I don't know what Terminal is and I'm nervous about running commands. Please walk me through the full first-time setup. I need you to: 1. Install the project's dependencies (run npm install for me) 2. Install the GitHub CLI (gh) if it's not already installed 3. Help me authenticate with GitHub — I'll do the browser sign-in part, you tell me exactly what to click 4. Create a new PRIVATE GitHub repo for me, named "harmony-health-site" 5. Push all the code in this folder to that new GitHub repo Ask me to approve each command before you run it. Tell me in plain English what each command does before you run it. If anything fails, explain what went wrong and how to fix it in simple terms. When you're finished, give me the GitHub URL of my new repo so I can use it to connect to Vercel in the next step.
What happens next Claude reads the folder, starts running commands one at a time, and asks the team member to click Allow (or similar) before each one. Claude will:
  • Run npm install (1-2 min)
  • Check for Homebrew; if missing, install it or tell them to use the .pkg installer for gh
  • Run gh auth login — this opens a browser window where they click Authorize once
  • Run gh repo create harmony-health-site --private --source . --push — creates the repo AND pushes code in one command
  • Report back with the repo URL

3.4 Fallback — if anything gets weird

The most likely failure point is GitHub auth. If Claude says it can't auth, have them paste this:

The GitHub auth step isn't working. Can you try a different approach — maybe install gh differently, or use git's built-in credential manager? Walk me through whatever will work.

If Node.js isn't installed (npm install fails), Claude will tell them. Have them install it manually from:

https://nodejs.org/en/download

Download the LTS version, Mac .pkg installer. Restart VS Code after. Then have Claude retry.

3.5 Confirm it worked

  1. Claude gives them the GitHub URL (copy it somewhere — they'll need it in Part 4)
  2. Have them open the URL in a browser. They should see all the site files.
That's the last time Claude runs setup commands From now on, Claude only runs git commit and git push when they say "publish." Setup is a one-time thing.

4 Connect GitHub to Vercel (5 min)

  1. Go to:
    https://vercel.com/new
  2. Their GitHub repos appear. Click Import next to harmony-health-site.
  3. Vercel auto-detects Next.js. Leave all settings at defaults.
  4. No environment variables needed.
  5. Click Deploy.
  6. Wait ~90 seconds. First deploy goes live at something like harmony-health-site-abc123.vercel.app.
  7. Open the URL. Confirm the site renders.

5 Point GoDaddy at Vercel (10 min + wait)

Critical Harmony Health's email is Google Workspace. Do NOT change nameservers. Only change the A record and CNAME — this preserves MX records and keeps email working.

5.1 In Vercel

  1. Go to the new project → Settings → Domains
  2. Enter yourharmonyhealth.com → click Add
  3. Vercel shows the required DNS config. Confirm it matches the table below.
  4. Also add www.yourharmonyhealth.com (Vercel auto-redirects one to the other)

5.2 In GoDaddy

GoDaddy → Domain Portfolio → click yourharmonyhealth.com → DNS → Manage Zones.

Copy-ready DNS values:

76.76.21.21
cname.vercel-dns.com
ActionTypeNameValue
Edit existing A @ Change from 160.153.0.74 to 76.76.21.21
Edit or add CNAME www Set to cname.vercel-dns.com

Leave untouched: MX records (Google Workspace), SPF/TXT records, nameservers.

5.3 Wait for DNS to propagate

  • Usually 10–30 minutes
  • Up to 48 hours worst case
  • Check in a private browser window. Refresh every few minutes.
  • Vercel's Domain page shows a green checkmark when ready
  • SSL auto-issues once DNS resolves — no action needed

6 Desktop shortcut launcher (3 min)claude does it

Goal: double-click the Desktop icon → VS Code opens to the project → Claude chat is ready. Claude creates the shortcut for them.

6.1 Paste this prompt into Claude

In the Claude chat panel (same one they've been using), paste:

Create a Desktop shortcut called "Harmony Health editing" that opens this project folder in VS Code when I double-click it. I'm on a Mac. I don't know what Terminal is. Please handle everything — creating the file, making it executable, and placing it on my Desktop. Use ~/harmony-health as the folder path so it works no matter what my Mac username is. When done, tell me it's ready to test.

Claude will:

  • Create Harmony Health editing.command on their Desktop
  • Make it executable (no user action needed)
  • Confirm it's ready

6.2 Test it

  1. Go to their Desktop (minimize VS Code, or ⌘+F3 to show Desktop)
  2. Double-click Harmony Health editing
  3. A small black Terminal window flashes open briefly, then VS Code opens to the project folder
  4. Click the Claude icon in VS Code's left sidebar — the chat is ready
First-time permission popup On first double-click, macOS may ask to confirm because the file came from an unknown source. Click Open. It won't ask again.

6.3 (Optional) Give it a Harmony Health icon

  1. Find any Harmony Health logo image on your computer (a .png or .jpg)
  2. Double-click to open it in Preview
  3. Press ⌘+A to select all, then ⌘+C to copy
  4. On the Desktop, single-click the Harmony Health editing file to highlight it
  5. Press ⌘+I to open Get Info
  6. Single-click the little icon in the top-left corner of the Get Info window (it will highlight blue)
  7. Press ⌘+V to paste the logo
  8. Close the Get Info window. The shortcut now has a branded icon.

Easiest way — they drag it in Finder:

8 Practice edit (5 min)

Walk them through the full loop end-to-end so they see how easy it is.

  1. They double-click Harmony Health editing on Desktop
  2. VS Code opens. Click the Claude icon in the left sidebar if the chat panel isn't already showing.
  3. Paste this into the Claude chat box:
    Change the tagline "Personalized care led by Megan Cryder, NP-C" at the bottom of the homepage hero to "Personalized care led by Megan Cryder, NP-C · Est. 2024." Then publish it.
  4. Press Return. Claude finds the file, makes the change, and shows them the diff.
  5. Claude may ask for permission to commit + push. Click Allow.
  6. Wait 90 seconds. Refresh yourharmonyhealth.com in a private browser window.
  7. They see their change live.
  8. Have them reverse it:
    Actually, change it back to "Personalized care led by Megan Cryder, NP-C" and publish.

Now they've seen the full editing loop. They know it works.

After the call

Send them an email linking to:

  • Chase's phone number for when things get weird

Rollback policy

Never auto-rollback
  • Vercel auto-serves the last working version until a new build succeeds. A failed build cannot replace a live site.
  • If a deploy succeeds but looks wrong, the team member should text Chase. Do NOT have Claude make more edits to fix it.
  • Chase rolls back via Vercel dashboard → Deployments → prior working one → Promote to Production.
